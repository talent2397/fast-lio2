### auto_explorer v17 当前架构

```
传感器管线:
  /robot/lidar (Gazebo原始, 3600采样, 10Hz)
    → cloud_to_scan v5 → /scan (710/720rays, robot/base_link帧)
      ├─→ slam_toolbox → /map → 前沿检测
      └─→ auto_explorer 直接读 /scan → APF势场避障

参数:
  MAX_V=1.2 MAX_W=2.0 K_REP=1.0 K_ATT=1.5 K_ANG=1.8 D_SAFE=2.0 D_BACK=1.0
  W_GAIN=2.0 W_DIST=0.4 W_TURN=1.5
  PRE_BRAKE=1.1m HARD_BRAKE=0.8m EMERGENCY=0.4m LOS_MAX=8m FT_DIST_DENOM=45

安全三层:
  1. PRE_BRAKE 1.1m (窄锥±20°): speed_limit=0.25~MAX_V
  2. HARD_BRAKE 0.8m (宽锥±45°): 后退1.2s → post_brake 2.5s → force_straight 4s
  3. EMERGENCY 0.4m: 全停, lin=-0.3

状态机 (优先级从高到低):
  1. 硬刹车后退 (SPIN_WAIT不触发)
  2. 安全检查 _safety_check (永不被屏蔽)
  3. post_brake 逃脱转弯 (v17: _post_brake_full防半程bug)
  4. ACQUIRE: YOLO检测到目标但无3D → 慢转+不累积stuck, 10s超时→5s冷却
  5. →CHASE: 有检测有3D → APF追踪到目标前2m
  6. SPIN_WAIT: 旋转1.2圈+map>5%, 超时20s
  7. CHASE: APF追踪融合目标
  8. REACHED: 到达停机
  9. EXPLORE核心:
     a. spin-trap检测: 高角速+低线速>2.5s → force_straight
     b. thrashing检测: yaw翻转>5次/1.5s → force_straight
     c. force_straight: 最低0.3m/s, 完整APF转向
     d. 前沿路径跟随: LOS→APF, stuck>5s放弃
     e. 前沿规划: 聚类(采样i%2+BFS±3, min=5) → 打分 → A*多备选(top5)
     f. stuck处理: esc<3→post_brake 2s→force_straight 5s; esc≥3→大撤退8s
     g. 无前沿: push(5-20)→spin-scan(≥20, 8s清vst, 12s逃脱5s)
  10. 兜底: 避_bias_angle推

前沿聚类 (v16修复):
  步长 i%2 + BFS ±3邻域 + 最小5像素(~0.5m宽门缝)

A*规划 (v16修复):
  前5名前沿依次尝试, 任一成功即采用

APF势场 (v16修复):
  单点斥力截断 max(k)=5.0, 总排斥力截断 max_rep=8.0

禁区_vst (v16修复):
  1.5m×1.5m网格, hard_brake/post_brake/到达前沿/A*失败/stuck esc≥3时标记
  spin-scan每8s清, scan_push完成清, 所有前沿被拉黑时自动清
```
