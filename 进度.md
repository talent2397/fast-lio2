  1. 硬刹车后退 (SPIN_WAIT不触发)
  2. 安全检查 _safety_check (永不被屏蔽)
  3. post_brake 逃脱转弯 (v17: _post_brake_full防半程bug)

  6. SPIN_WAIT: 旋转1.2圈+map>5%, 超时20s
  7. CHASE: APF追踪融合目标
  8. REACHED: 到达停机
  9. EXPLORE核心:
     a. spin-trap检测: 高角速+低线速>2.5s → force_straight
     b. thrashing检测: yaw翻转>5次/1.5s → force_straight
     c. force_straight: 最低0.3m/s, 完整APF转向
     d. 前沿路径跟随: LOS→APF, stuck>5s放弃
     e. 前沿规划: 聚类(采样i%2+BFS±3, min=5) → 打分 → A*多备选(top5)
     f. stuck处理: esc<3→post_brake 2s→force_straight 5s; esc≥3→大撤退8s
     g. 无前沿: push(5-20)→spin-scan(≥20, 8s清vst, 12s逃脱5s)
  10. 兜底: 避_bias_angle推

前沿聚类 (v16修复):
  步长 i%2 + BFS ±3邻域 + 最小5像素(~0.5m宽门缝)

A*规划 (v16修复):
  前5名前沿依次尝试, 任一成功即采用

APF势场 (v16修复):
  单点斥力截断 max(k)=5.0, 总排斥力截断 max_rep=8.0

### v18 CHASE 修复详情 (2026-07-24)

**根因分析**:
1. `det_fresh` 超时仅4s, YOLO ~2Hz检测+转弯遮挡 → CHASE快速丢失
2. CHASE→EXPLORE 切换时 stuck 已累积 → 立即触发逃离 → 离球更远
3. APF 局部极小值——柱子前引力+斥力抵消 → 绕圈不前进
4. HARD BRAKE 后 force_straight 仅EXPLORE处理 → CHASE模式下被忽略

**修复**:
| 项目 | 变更 |
|------|------|
| `det_fresh` 阈值 | 4s → 10s |
| CHASE丢失宽限期 | 10s (仅YOLO丢但fusion鲜时保持CHASE) |
| fusion保鲜 | 20s (fusion_tgt最后更新时间) |
| CHASE stuck | 每tick递减, 不累积, escalate保持0 |
| CHASE APF | 新 `_potential_field_chase`: K_ATT×1.7, max_rep 5.0, 后半球斥力×0.3 |
| CHASE障碍绕行 | `_map_ready()+F<1.5m` → A* 规划绕行路径 + LOS前瞻 |
| force_straight | 移至状态机之前(通用), CHASE时朝向目标 |
| post_brake→CHASE | force_straight方向朝向fusion目标 |
| HARD BRAKE | 清 `_chase_plan` 强制重规划 |
| CHASE→EXPLORE | 重置 `_bias_angle`, `_no_ft_count` |
| WASD终端 | 从 `start_sim.sh` 移除 |

## ✅ Fusion v7.1 + 简化世界 (2026-07-24)

### 核心问题诊断

v6/v7初期存在**3D定位系统误差**，导致机器人追幻影目标，停在错误位置：

| 问题 | 根因 | 影响 |
|------|------|------|
| 坐标系变换错误 | `_cam_to_odom()` 光学帧→机器人帧变换矩阵写反 | 目标出现在完全错误的方向 |
| FAST-LIO2 TF vs Gazebo odom | Fusion用FAST-LIO2的TF, auto_explorer用Gazebo odom | 两个坐标系不匹配, 累积偏差 |
| 地面三角测量被移除 | v7改成了球心三角测量, bbox像素Y误差放大 | 目标距离测量比真实值大30% |
