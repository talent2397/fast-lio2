## 📝 遇到的问题与解决方案

| # | 日期 | 问题 | 原因 | 解决方案 |
|---|------|------|------|---------|
| 1 | 2026-07-15 | 键盘控制小车不动 | bridge 桥接到全局 `/cmd_vel`，但 DiffDrive 监听 `/model/diff_car/cmd_vel` | 修改 `gz_bridge.yaml` 话题名 + 去掉 SDF 中 `<topic>` 声明 |
| 2 | 2026-07-15 | RViz 报 `No transform from [diff_car/chassis] to [camera_init]` | TF 树断裂：Gazebo 发布 `diff_car/odom→chassis`，但缺 `camera_init→odom` 和 `chassis→body` | 添加两个静态 TF 发布者 |
| 3 | 2026-07-15 | `/odom` 话题无数据 | bridge 缺少 `/model/diff_car/odometry` 桥接 | 添加 odom + tf 桥接到 `gz_bridge.yaml` |
| 4 | 2026-07-15 | 键盘控制不稳定（W/S慢、方向反） | Python 版 150ms 超时覆盖不了 660ms 按键重复延迟；A/D 方向写反 | 改为两段超时（700ms 初按 + 200ms 重复）+ 交换 A/D |
| 5 | 2026-07-15 | GitHub Push 超时 | 128MB 的文档图片（PDF/GIF/PNG）太大 | 排除 `doc/`、`rviz_cfg/`，从 85 文件减至 54 文件 |
| 6 | 2026-07-15 | Token 权限不足 (403) | Token 仅有 `metadata=read` 权限 | 换用 `repo` 权限的 classic token |
| 7 | 2026-07-15 | SSH 公钥不匹配 | 生成和显示用的是两个不同的公钥 | 重新复制正确的公钥到 GitHub |
| 8 | 2026-07-15 | 仿真参数落后 | 线速度 1.0 m/s、角速度 0.8 rad/s 太慢 | 改为线速度 3.0 m/s、角速度 2.0 rad/s |
| 9 | 2026-07-15 | 包名和话题命名不规范 | `fastlio_sim` 名字不统一，传感器 topic 为 `/lidar0` `/imu0` 无法适配真机迁移 | 重构为 `robot_bringup` + `robot_navigation` + `robot_vision`，统一话题 `/robot/*` |
| 10 | 2026-07-15 | 缺少相机传感器 | SDF 模型只有 LiDAR + IMU，YOLO 需要图像输入 | 添加 camera_link + camera 传感器，topic `/robot/camera`，640×480 30fps |
| 11 | 2026-07-15 | 车轮建模方向错误 | 四轮都用 sphere、用双旋转扭歪关节轴、两驱动轮两从动轮大小不一 | 改为圆柱轮子：roll=-1.5707只旋转几何体不旋转link、关节轴保持0 1 0、四轮统一r=0.2、Y=±0.30贴车身两侧 |
| 12 | 2026-07-15 | 小车侧翻 | mu=100摩擦力过大、COM太高、加速度太大 | mu→1.0、车身Z 0.55→0.20、加速度0.8m/s²、四轮加关节阻尼 |
| 13 | 2026-07-15 | Gazebo 闪退 | XML中重复 `<model name="robot">` 标签 | 检查SDF标签匹配，去重 |
| 14 | 2026-07-15 | 键盘控制闪退/卡顿 | 多种重写方案失败: curses/bscan/threads | 最终方案: 线程+800ms超时+自适应两段(750ms→180ms) |
| 15 | 2026-07-15 | YOLO无法安装 | pip安装torch ~2GB太慢 | HSV模式先行: 色域分割检测红色球体，`vision_msgs/Pose2D`字段测试通过 |
| 16 | 2026-07-15 | HSV阈值过严 | Gazebo 渲染饱和度低 (max S=113)，原阈值 S>100 过滤全部红色 | 降为 S>30 V>40，红球从(12,5)→(5,0) 近处便于检测 |
| 17 | 2026-07-20 | YOLO 安装后 numpy 2.x 不兼容 matplotlib | ultralytics 依赖 numpy>=2，但系统 matplotlib 基于 numpy 1.x 编译 | `pip install "numpy<2"` 降级，YOLO+HSV 双模正常 |
| 18 | 2026-07-20 | colcon 编译失败：setuptools `--uninstall` 选项不识别 | ultralytics 安装时升级 setuptools 到 83.0.0，colcon-core 要求 <80 | `pip install "setuptools<80"` 降级到 79.0.1 |
