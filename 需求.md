# 球式机器人 — 项目需求文档

## 1. 项目概述

设计并实现一个**球式滚动机器人**，能够在**陌生的室内环境**中自主完成：
- LiDAR-惯性 SLAM 定位建图（FAST-LIO2）
- YOLO 视觉目标识别（红色球体 / 黑色椅子）
- Nav2 自主导航至目标前方 2m 范围内停止

全程无人干预，纯自主完成任务。

---

## 2. 硬件平台

| 组件 | 型号 | 职责 |
|------|------|------|
| 机载电脑 | NVIDIA Jetson AGX Orin | SLAM + 视觉推理 |
| 协处理器 | 树莓派 Pi 5 | 电机控制 + 传感器采集 |
| LiDAR | RoboSense Airy | 3D 点云 |
| 摄像头 | IMX415 800W | 目标识别 |
| IMU | H30 | 惯性测量 |
| 底盘 | 球式滚动 | 全向移动 |

**通信架构：**

```
传感器 (LiDAR/IMU/Camera) → Jetson AGX (ROS 2)
                                ↓ (以太网 / ROS 2 多机)
                          树莓派 Pi 5 → 电机驱动
```

## 4. 任务流程（状态机）

```
         ┌─────────────────────────────────┐
         │                                 │
    ┌────────┐   ┌──────────┐   ┌───────┐ │
    │  INIT  │──▶│ EXPLORE  │──▶│VERIFY │─┤
    │  初始化 │   │ 边探索   │   │确认3D │ │
    └────────┘   │ 边建图   │   │ 坐标  │ │
                 │ + YOLO   │   └───┬───┘ │
                 └────┬─────┘       │     │
                      │  未找到     ▼     │
                      │         ┌─────────┐│
                      │         │ NOT_FOUND│
                      │         │ 日志输出 │
                      │         │ 等待指令 │
                      │         └─────────┘
                      │  找到目标
                      ▼
                 ┌──────────┐
                 │ NAVIGATE │
                 │ Nav2 导航 │
                 │ 到目标前  │
                 │ 方 2m    │
                 └────┬─────┘
                      ▼
                 ┌──────────┐
                 │ REACHED  │
                 │ 到达停止  │
                 └──────────┘
```

### 状态说明

| 状态 | 动作 |
|------|------|
| INIT | 加载配置、初始化传感器、等待 FAST-LIO2 首帧 |
| EXPLORE | nav2_explore_lite 驱动探索 + 每帧 YOLO 检测 + slam_toolbox 建图 |
| VERIFY | YOLO 检测到目标 → LiDAR 点云投射获取 3D 坐标 → 发布目标点 |
| NOT_FOUND | 全环境探索完毕未找到 → 输出日志、等待指令 |
| NAVIGATE | Nav2 路径规划 + 避障导航到目标点前方 2m |
| REACHED | 到达目标范围 → 停机、输出日志 |

## 5. YOLO + SLAM 融合方案

### 5.1 3D 定位管线

```
Camera 图像 → YOLOv8 检测 → 2D bounding box
                                   ↓
同步的 LiDAR 点云 ──→ 投影到图像平面 (内外参标定)
                                   ↓
                         取 bbox 内点云 3D 坐标的均值/中值
                                   ↓
                        输出目标 3D 世界坐标 → Nav2 goal
```

### 5.2 识别目标

| 目标 | 检测方式 | 备注 |
|------|---------|------|
| 红色球体 | 颜色分割 + 形状识别 + YOLO 分类 | 可同时使用 HSV 颜色过滤 |
| 黑色椅子 | YOLO 物体检测 | COCO 预训练权重已有 chair 类别 |

### 5.3 训练策略

1. 先用 COCO 预训练权重在仿真中跑通流程（COCO 包含 ball、chair）
2. 仿真验证通过后，在真实环境采集数据 → 微调模型
3. 开放数据集（如 COCO、OpenImages）同时适配仿真和真机环境

## 6. 导航方案

### 6.1 地图管线

```
FAST-LIO2 /cloud_registered (3D 点云)
    → pointcloud_to_laserscan → /scan (2D)
        → slam_toolbox (online_async, pose = FAST-LIO /Odometry)
            → /map (2D 占据栅格)
                → Nav2 global_costmap + local_costmap
```

### 6.2 Nav2 配置要点

- 全局规划器：SmacPlanner2D 或 NavFn
- 局部控制器：RPP（Regulated Pure Pursuit）或 DWB
- 代价地图：仅 2D，不需要高度信息
- 定位源：FAST-LIO2 里程计（高精度，无漂移问题）
- 探索：nav2_explore_lite（前沿边界驱动）

## 7. 仿真→真机迁移策略

### 7.1 核心原则

**话题名统一，接口隔离，配置切换**

仿真和真机使用完全相同的话题名。区别只在数据来源：

| 环节 | 仿真 | 真机 |
|------|------|------|
| LiDAR 数据 | Gazebo gpu_lidar → gz_bridge → `/robot/lidar` | RoboSense Airy 驱动 → `/robot/lidar` |
| IMU 数据 | Gazebo IMU sensor → gz_bridge → `/robot/imu` | H30 IMU 驱动 → `/robot/imu` |
| Camera 数据 | Gazebo camera → gz_bridge → `/robot/camera` | IMX415 GStreamer → `/robot/camera` |
| cmd_vel | Nav2 → gz_bridge → Gazebo DiffDrive | Nav2 → Pi5 电机驱动节点 |
| 启动方式 | `bash start_sim.sh` 或 `ros2 launch robot_bringup sim.launch.py` | `ros2 launch robot_bringup real.launch.py` |

### 7.2 标定（真机）

- LiDAR-Camera 外参标定（`lidar_camera_calibration` 或 APRIL tag）
- IMU-Camera 时间同步
- 轮式里程计标定（如需）
